# 🚀 自动化开发工作流 - 快速开始

> 一句话完成：编码 → 审查 → 修复

## ⚡ 最简单的用法

在 Claude Code 中直接说：

```
实现 [你的功能]，完成后自动用 Opus 审查，发现严重问题自动修复。
```

**就这么简单！** Claude 会自动：
1. ✅ 用 Sonnet 快速实现
2. ✅ 自动切换到 Opus 深度审查
3. ✅ 发现问题后自动修复
4. ✅ 给你完整的报告

## 📋 常用模板

### 1. 基础开发 + 审查

```
实现一个 [功能描述]

要求：
- 添加必要的测试
- 完成后自动代码审查
- 生成实现文档
```

### 2. 安全敏感功能

```
实现 [认证/支付/加密等功能]

要求：
- Sonnet 实现基础功能
- Opus 进行安全审查（OWASP Top 10）
- 有问题自动修复
- 生成安全审查报告
```

### 3. 性能关键功能

```
实现 [数据处理/API等功能]

要求：
- 实现后并行运行：
  - 性能审查
  - 代码质量审查
- 汇总所有建议
```

### 4. 增量开发

```
我要分多个阶段实现一个复杂功能。

Phase 1: [核心功能]
- 实现后审查，确认方向

Phase 2: [扩展功能]
- 实现后审查

每个阶段审查通过后再继续。
```

## 🎯 高级用法

### 并行审查（更快）

```
实现完成后，并行运行以下审查：
1. 安全审查 (Opus)
2. 性能审查 (Sonnet)
3. 测试覆盖率审查 (Sonnet)

汇总结果后一次性展示。
```

### 指定审查标准

```
按照以下标准审查：
- Google 代码审查标准
- 测试覆盖率 > 80%
- 无 Critical 安全问题
```

### 自动化 Git 工作流

```
实现 [功能]，完成后：
1. 审查代码
2. 如果通过，自动创建 commit
3. 生成 commit message
4. 显示 git diff 让我确认
```

## 💡 实用技巧

### 技巧 1: 保持上下文

Claude Code 会记住你的偏好，一次设置后：

```
记住：以后我让你实现功能时，都自动进行代码审查。
```

之后只需说：
```
实现 XXX 功能
```

Claude 会自动审查。

### 技巧 2: 自定义审查级别

```
设置审查级别：
- Level 1 (快速): 只检查明显错误
- Level 2 (标准): 代码质量 + 安全
- Level 3 (严格): 包括代码风格和最佳实践

默认使用 Level 2，我说 "严格审查" 时用 Level 3。
```

### 技巧 3: 项目专属配置

在项目根目录创建 `.claude.json`:

```json
{
  "workflow": "auto-review",
  "review_standards": ["security", "performance", "testing"],
  "auto_fix": true,
  "model_preference": {
    "implementation": "sonnet",
    "review": "opus",
    "fix": "sonnet"
  }
}
```

然后：
```
按照项目配置实现 [功能]
```

## 🔥 真实案例

### 案例 1: API 开发

**传统方式:**
1. 在 Claude 中: "帮我实现一个用户登录 API"
2. 得到代码
3. 新建对话，切换到 Opus
4. "帮我审查这段代码：[粘贴代码]"
5. 得到审查意见
6. 回到原对话或新开 Sonnet 对话
7. "根据这些意见修复：[粘贴意见]"

**自动化方式:**
1. "实现用户登录 API，完成后自动审查并修复问题"
2. 等待完成，查看结果

### 案例 2: Bug 修复

**自动化方式:**
```
修复 [Bug 描述]

要求：
1. 分析根本原因
2. 实现修复
3. 添加测试防止回归
4. 审查修复方案是否合理
5. 确认没有引入新问题
```

一句话完成整个流程！
